NEWS-MLOPS: Retrieval-Augmented News Query System
MLOps Exam Project – MSc BDS Data Engineering and Machine Learning Operations in Business
Author: Riya Pokharel
1. Project Overview
This project implements an end‑to‑end MLOps pipeline for a news‑based Retrieval-Augmented Generation (RAG) system.
The system ingests news data, generates embeddings, stores them in a ChromaDB vector database, retrieves relevant documents, and exposes an API endpoint for querying the system.

This project demonstrates:

Data ingestion and preprocessing

Embedding generation

Vector database creation (ChromaDB)

Retrieval and RAG pipeline

API deployment using FastAPI

Artifact tracking

Containerization with Docker

3. Pipeline Description
3.1 Data Ingestion
create_data.py generates data.json, which is the dataset used by the system.
This file is intentionally kept outside the artifacts/ folder because the API loads it directly.

3.2 Embedding and Vector Store
vector_db.py generates embeddings from data.json and stores them in a ChromaDB vector database.
The vector store is saved inside the chroma_db/ directory.

3.3 Retrieval-Augmented Generation
rag.py performs:

Retrieve relevant documents from ChromaDB

Combine retrieved context with the user query

Generate a final answer using an LLM

3.4 API Deployment
app_fastapi.py exposes a single API endpoint:

Code
POST /ask
Input: user query
Output: model‑generated answer with retrieved context

This is the deployment scenario implemented for the exam.

4. Running the Project Locally
4.1 Install Dependencies
Code
pip install -r requirements.txt
4.2 Generate Data
Code
python create_data.py
4.3 Build Vector Database
Code
python vector_db.py
4.4 Start the API
Code
uvicorn app_fastapi:app --host 0.0.0.0 --port 8000
API documentation will be available at:

Code
http://localhost:8000/docs
5. Running with Docker
5.1 Build the Docker Image
Code
docker build -t news-backend .
5.2 Run the Container
Code
docker run -p 8000:8000 news-backend
API will be available at:

Code
http://localhost:8000/docs
